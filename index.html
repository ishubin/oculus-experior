Oculus Experior Introduction

Oculus Experior is a testing framework designed with simplicity and flexibility in mind. It gives you a lot of comprehensive features which help you to create robust tests with clean code.

Take a look at some of the features:

  • Test Dependencies In your test suite you can specify the dependencies between tests. If first test fails - the dependent test will be skiped.
  • Parameter Dependencies Not only you can specify dependency between tests, but also you can configure dependency between parameters of different tests.
  • Data providers Gives you an ability to abstract all data initialization and design a clean test code.
  • Structured Reports You can structure your reports to make them more readable.
  • Event Listeners You can assign listeners for different events in your tests. There is quite a plenty of events available so you can modify the test logic in varied ways
  • Rollbacks Allows you to go back through your test actions and invoke "undo" methods for each specific action. This can be used for production testing for cleaning up the data generated by test.
  • Error Handling Allows to handle exceptions just generally within test and also for any specific action separately.
  • Integration with Frontend And last but not least extremely useful feature which allows you to make your test report available through a web interface. For this you also need Oculus Frontend webapp.
