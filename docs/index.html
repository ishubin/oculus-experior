Oculus Experior Introduction

Oculus Experior is a testing framework which is mainly designed to ease work on development of Selenium tests and provide flexibility on setting the test logic.
It supports the following features:
  • Test Dependencies In your test suite you can specify the dependency of tests. So in case if first test fails - the dependent test will be skiped.
  • Parameter Dependencies Not only you can specify dependency between tests, but also you can specify that some test parameters depend on parameters of other tests.
  • Structured Reports You may design you reports in a structured (tree) way and make it more readable.
  • Event Listeners You can assign handlers for different events in your tests. There is quite a plenty of events available so you can modify the test logic in varied ways
  • Rollbacks Allows you to go back through your test actions and invoke handlers for each action separately. This can be used for production testing for cleaning up the data generated by test.
